# 4 何时进化

基于大语言模型的智能体中，自进化的 temporal dimension（时间维度）主要关注学习过程与任务执行之间的关系。因此，自进化智能体的第二个关键方面是识别进化时机（evolving timing），即自进化策略 $f$ 在哪个阶段被调用并应用于智能体系统。为此，我们提出了一种分类法，区分两种时间模式的自进化：**Intra-test-time self-evolution（测试内自进化）** 和 **Inter-test-time self-evolution（测试间自进化）**。

**Intra-test-time self-evolution（测试内自进化）** 指的是发生在任务执行过程中的自适应过程，智能体在这一过程中识别出其在特定问题上的局限性，并启动有针对性的学习机制，以实时（real-time）方式增强其能力（Xi et al., 2024; Bi et al., 2024）。这种进化模式的特征在于与当前任务的即时耦合：智能体针对所遇到的特定问题改进其问题求解能力，在性能与适应之间形成动态互动。

**Inter-test-time self-evolution（测试间自进化）** 指的是发生在任务完成之间的学习过程，利用积累的经验来提升未来性能。这一类别涵盖多种方法论路径：离线学习范式（offline learning paradigm），通过迭代精炼从预收集的数据集中提取知识（Zelikman et al., 2022; 2024）；以及在线学习范式（online learning paradigm），基于流式交互数据持续地进行自适应（Qi et al., 2024; Qiu et al., 2025b; Qian et al., 2024b; Wang et al., 2025o）。

自进化在这些时间阶段的实现利用了大语言模型中的三种基本学习范式：

- **In-context learning（上下文学习，ICL）**（Dong et al., 2022; Min et al., 2021; Wies et al., 2023），通过上下文中的示例在不修改参数的情况下自适应行为；
- **Supervised Fine-Tuning（有监督微调，SFT）**，通过对标注数据进行基于梯度的优化来更新模型权重（Devlin et al., 2018; Shen, 2024; Dong et al., 2023）；
- **Reinforcement Learning（强化学习，RL）**，通过奖励驱动的策略优化来塑造行为（Kaelbling et al., 1996; Sun et al., 2024a; Zhang et al., 2025n）。

虽然这些学习范式在时间上下文之间保持着概念上的一致性，但它们在数据可用性和学习目标方面的具体实现有所不同：

**Intra-test-time（测试内）** 的特点是其在线特性：学习数据在任务执行过程中动态涌现，优化直接针对即时问题实例的性能提升。这种实时耦合需要能够快速处理学习数据和反馈信号、并在对活跃任务求解的时间限制内修改行为的快速自适应机制。

另一方面，**Inter-test-time（测试间）** 的特点是其回溯特性：学习算法对历史数据进行操作，无论是来自精心整理的数据集还是积累的行为轨迹，优化目标面向提升任务分布上的预期性能，而非最大化任何特定问题实例上的成功率。这种时间上的解耦使得更复杂的学习过程能够识别跨任务模式、整合多样化经验，并在活跃任务执行的即时性约束之外发展出可泛化的能力。

## 4.1 Intra-Test-Time Self-Evolution（测试内自进化）

在测试内自进化中，智能体参与与解决当前任务内在耦合的自我改进过程。这一时间阶段的显著特征是其同步性：反馈信号在任务执行过程中生成和处理，优化目标专门针对当前问题实例的性能提升，而非泛化到未来任务。在此，我们介绍三种学习范式在这一时间阶段中的具体实现。

### In-Context Learning（上下文学习）

测试内 ICL 方法利用模型的上下文窗口作为即时适应的动态记忆系统，无需修改参数。这些方法通常采用自反思机制（self-reflective mechanism），智能体分析自身表现，生成文字形式的批评或见解，并将这些反思保存在情景记忆缓冲区中，以在同一任务上下文中指导后续决策（Shinn et al., 2023; Madaan et al., 2023a）。一些方法超越了简单的反思，包含动态规划修订（dynamic planning revision），智能体可以根据环境反馈修改其整体方法，根据需要在动作执行和计划修改之间切换。例如，AdaPlanner（Sun et al., 2023）将任务分解为可管理的子目标，并为每个子目标预测环境反馈。在执行过程中，其优化器组件（refiner component）区分计划内反馈（与预测一致的观察）和计划外反馈（偏离预测的观察）。对于计划内反馈，优化器通过专门的 `ask_LLM()` 动作动态查询大语言模型，以解析观察结果并提取相关信息。对于计划外反馈，优化器主动修订整个计划，并从中间点恢复求解，而非从头开始。这种自适应闭环框架无需预先了解反馈结构，并实现了更高效的决策。类似地，TrustAgent（Hua et al., 2024）在执行过程中采用基于规则的计划修订，根据语言反馈修改其方法，逐步演化为更安全的规划策略。这些 ICL 方法展示了测试时适应如何实现复杂的行为修改而无需对模型进行永久性更改，在保持灵活性的同时保留了模型的通用能力。

### Supervised Fine-Tuning（有监督微调）

测试内 SFT 代表了一种范式转变，模型通过习得的元适应策略执行即时自我修改。自适应语言建模（Zweiger et al., 2025）是这一方法的典型代表，它生成"自我编辑"（self-edits），即元层级的指令，可以重组信息表示、指定优化超参数，或调用工具进行数据增强和梯度计算。这些自我编辑触发即时有监督微调，产生持久的权重更新，使模型适应当前任务。其关键创新在于元学习阶段，强化学习使用更新后模型的下游性能作为奖励信号来训练模型产生有效的自我编辑，本质上是教导模型如何自我教学。Acikgoz 等人（2025）引入了一种测试时自我改进（Test-Time Self-Improvement，TT-SI）框架，使智能体能够通过自我意识首先识别不确定的测试样本。对于这些具有挑战性的输入，智能体随后生成一个综合训练样本，并执行一次临时、轻量级的参数更新，以在重置权重之前提升其即时性能。

### Reinforcement Learning（强化学习）

测试内 RL 使模型能够在遇到超出其当前能力范围的问题时按需开发新能力。LADDER（Simonds & Yoshiyama, 2025）通过其测试时强化学习（Test-Time Reinforcement Learning，TTRL）机制展示了这一点：一旦识别出特别具有挑战性的问题，系统生成一组聚焦的相关问题变体，并针对该问题类别进行集中、有针对性的强化学习。这种方法将难以逾越的挑战转化为学习机会，使模型能够在部署期间扩展其问题求解技能库，而非失败或提供次优解。该方法代表了一种即时技能获取（just-in-time skill acquisition）形式，计算资源精确地在最需要的时间和地点投入。

## 4.2 Inter-Test-Time Self-Evolution（测试间自进化）

测试间自进化代表了自主智能体中主要的学习过程，发生在任务执行之后而非执行过程中。在这一时间模式下，智能体完成给定任务，提取反馈信号，包括显式奖励（Gao et al., 2024）、梯度（Amari, 1993; Bottou, 2010）和性能指标（Ge et al., 2023），随后利用这些信息来增强其未来问题求解的能力。这种回溯式学习过程将任务性能与能力提升解耦，使智能体能够整合经验、识别成功与失败的模式，并在实时任务需求施加的计算约束之外系统地优化其行为策略。

### In-Context Learning（上下文学习）

测试间上下文学习已成为智能体自我改进的广泛采用方法。该范式将先前任务的执行结果和反馈作为未来问题求解的上下文信息加以利用。Wang 等人（Wang et al., 2024j）通过从智能体动作历史中归纳工作流并将其纳入后续任务的上下文来验证这一原则。上 Reinforcement Learning（上下文强化学习，ICRL）领域（Moeini et al., 2025; Laskin et al., 2022; Lee et al., 2023）将这一概念扩展为在智能体上下文窗口中维护观察和动作的历史。这些方法利用了一个假设：预训练的神经网络可以在前向传播中实现隐式强化学习算法，处理上下文信息以在不进行参数更新的情况下自适应行为（Kirsch et al., 2023）。ICRL 的一个定义性特征是上下文内改进（in-context improvement）：即随着任务相关信息在上下文中积累，智能体性能逐步提升的现象，使通过注意力机制而非基于梯度的学习实现复杂适应成为可能。

### Supervised Fine-Tuning（有监督微调）

测试间 SFT（Chen et al., 2025b）方法建立了一种通过综合数据生成和自评估实现迭代自我改进的范式。SELF（Lu et al., 2023）开创了元认知训练（meta-cognitive training），模型首先获得自我反馈和自我改进能力，然后迭代地为未标注的指令生成响应，并通过自我批评来增强它们。STaR（Zelikman et al., 2022）和 Quiet-STaR（Zelikman et al., 2024）专注于通过理性化（rationalization）改进推理能力——模型尝试解决问题，然后为其最初未能正确解答的正确答案生成解释，创建将成功尝试与事后推理相结合的增强训练数据。SiriuS（Zhao et al., 2025b）将其扩展到顺序问题求解，维护正确解决方案的知识库，同时通过涉及反馈整合、再生和改述的多阶段精炼来增强失败案例。这些方法共享一个核心洞察：模型可以通过学习评估和增强自身输出来引导自我改进，从最初不完美的尝试中创建高质量的训练信号，而无需大量人工监督。ARIA（He et al., 2025a）等最新框架进一步将人工介入指导（human-in-the-loop guidance）纳入测试时适应，使智能体能够主动识别知识差距并请求专家反馈。

### Reinforcement Learning（强化学习）

测试间 RL 利用无约束的计算资源，通过广泛的环境交互和复杂的课程设计来优化智能体。RAGEN（Wang et al., 2025q）和 DYSTIL（Wang et al., 2025c）采用在线强化学习处理多轮交互任务，在模拟对话中通过同策略学习（on-policy learning）持续优化策略。像人类一样学习（Learning Like Humans）（Zhang et al., 2025a）引入了具有自适应难度递进的认知启发式训练，结合同策略探索与离策略效率和专家示范以加速学习。领域特定应用展示了测试间 RL 的多功能性：WebRL（Qi et al., 2024）通过根据性能自动调整任务复杂度的自进化课程来开发 Web 导航智能体，而 DigiRL（Bai et al., 2024）使设备控制智能体能够通过自主强化学习掌握真实环境中的交互。这些方法利用部署前的阶段进行大量的试错学习，通过数千次交互开发出稳健的策略，而这些交互在实时部署期间是不切实际的。
