## 7.2 评估范式（Evaluation Paradigm）

对自进化智能体的评估，考虑到其持续学习的范式，需要一种超越传统静态评估的多维度方法。当前的评估范式可根据评估的时间范围大致分为：静态评估（Static Assessment）、短时域自适应评估（Short-horizon Adaptive Assessment）和长时域终身学习能力评估（Long-horizon Lifelong Learning Ability Assessment）。每个类别关注智能体进化能力的不同方面，从即时性能到长期学习轨迹。术语说明：此处"持续（continuous）"指的是系统层面的自进化（跨回合的策略、记忆、技能、工具或流程的变化），并不一定指模型参数的持续学习。因此，我们的评估追踪跨交互回合及演化环境中的轨迹，无论权重是否更新。

### 7.2.1 静态评估

静态评估在特定时间点对自进化智能体的即时性能进行评估。尽管这些智能体被设计为持续改进，但静态方法对于建立基线性能、在固定任务集上比较不同智能体架构，或评估离散训练阶段后的能力，仍然至关重要。该方法与传统人工智能评估保持一致，侧重于固定环境中的即时性能。虽然对于"域内评估、域外评估"的泛化评估范式有用，但静态评估本质上映无法捕捉自进化智能体所核心的动态、持续学习或长期演化方面。

对于在给定时刻评估智能体的一般能力，通常采用为静态人工智能系统设计的标准基准测试。这些基准测试提供多样化的任务领域并测试各种核心智能体能力，提供智能体在其进化的特定阶段之前或之中的能力快照。这些评估可系统地分为外部任务解决评估（External Task-Solving Evaluation）和内部智能体组件评估（Internal Agent Components Evaluation），其中外部任务解决评估衡量在领域特定或跨领域任务中完成端到端性能，而内部能力评估则聚焦于智能体的基础组件，包括规划、工具利用、记忆管理、多智能体协作等。

**外部任务解决评估** 该类别评估智能体在各种现实或模拟环境中完成任务的端到端熟练程度。在科学数据分析与机器学习工程领域，ScienceAgentBench（Chen et al., 2024e）和 MLE-Bench（Chan et al., 2024）等基准测试考察智能体生成和执行数据分析代码以及解决 Kaggle 风格问题的能力。在网络搜索/浏览方面，WebShop（Yao et al., 2022）、WebArena（Zhou et al., 2023）、X-WebAgentBench（Wang et al., 2025j）、Mind2Web（Deng et al., 2023）和 BrowseComp（Wei et al., 2025b）等环境模拟逼真的网络交互、复杂浏览场景及安全约束下的任务完成。在软件工程领域，SWE-bench 系列（Jimenez et al., 2023; OpenAI, 2024; Aleithan et al., 2024; Yang et al., 2024）使用真实的 GitHub issue 来评估智能体的代码修复能力。对于计算机使用交互，OSWorld（Xie et al., 2024）提供了一个统一环境，用于处理涉及各种桌面和 Web 应用程序的开放性任务。营销等专业领域也有 xbench（Chen et al., 2025f）等基准测试。除特定领域外，AgentBench（Liu et al., 2023b）、GAIA（Mialon et al., 2023）和 TheAgentCompany（Xu et al., 2024b）等通用智能体基准测试评估跨多个知识领域和专业任务的广泛问题解决能力，模拟对通用人工智能助手的现实需求。

**内部智能体组件评估** 除了端到端任务完成之外，评估智能体的底层核心能力也至关重要。这些基准测试评估对智能体整体智能和自进化潜力有所贡献的基础能力。在**规划**方面，PlanBench（Valmeekam et al., 2023）、Natural Plan（Zheng et al., 2024a）、AutoPlanBench（Stein et al., 2025）和 ACPBench（Kokel et al., 2025）等基准测试全面评估智能体在各种规划领域中理解动态环境、制定策略、分解复杂问题和执行推理的能力。在**工具使用**方面，ToolAlpaca（Tang et al., 2023）和 ToolBench（Qin et al., 2023）等简单基准测试考察基本的选择和参数映射能力，而 ToolSandbox（Lu et al., 2024a）、Seal-Tools（Wu et al., 2024）、API-Bank（Li et al., 2023）、T-Eval（Chen et al., 2023）、$\tau$-Bench（Yao et al., 2024）、AceBench（Chen et al., 2025a）等更复杂的基准测试则模拟涉及多轮交互、隐式状态依赖和嵌套调用的现实场景。**记忆管理**基准测试如 LTMBenchmark（Castillo-Bolado et al., 2024）、MemoryAgentBench（Hu et al., 2025）和 StoryBench（Wan & Ma, 2025）评估智能体在多轮交互、动态场景和长距离依赖中保留和利用信息的能力。对于**多智能体协作**的评估，MultiAgentBench（Zhu et al., 2025）和 SwarmBench（Ruan et al., 2025）等基准测试评估协作与竞争设置中的协调、通信和涌现性群体智能。

静态评估的典型指标包括准确率（accuracy）、成功率（success rate）、进展率（progress rate）、完成率（completion rate）以及各种领域特定的性能指标（如 CodeBertScore、Valid Execution Rate、Pass Rate、F1 score）。这些指标为孤立的单次调用或固定任务集提供单一的性能分数。

### 7.2.2 短时域自适应评估

短时域自适应评估通过评估智能体在相对较短的时期内或有限的交互次数中适应和改进的能力，超越了静态评估。智能体可能通过更多尝试来提升同一任务实例上的表现，或适应同一类型任务的新实例。该类别侧重于捕捉自进化智能体在相对一致或轻微变化的任务分布中即时适应性和增量学习的能力。这些评估方案可大致分为两类：（1）为传统基准测试增加时间维度；（2）专门设计本质上支持短时域动态学习的基准测试和指标。

**增加时间维度的传统基准测试** 许多研究利用现有基准测试，但引入新维度以追踪随时间变化的性能。这通常涉及将性能作为迭代次数、步数或样本数的函数进行分析。例如，ADAS（Hu et al., 2024c）在 ARC 基准测试（Chollet, 2019）上评估了智能体系统迭代次数与留出测试准确率的关系；AWM（Wang et al., 2024j）在 WebArena 地图测试划分（Zhou et al., 2023）下研究了在线评估过程中随样本数量的累积成功率；WebEvolver（Fang et al., 2025b）研究了 Mind2web-Live（Pan et al., 2024）下自改进迭代与成功率的关系。这种方法允许在有限范围内追踪智能体的适应能力（Adaptivity）。

**内置动态评估的基准测试** 某些基准测试在设计时就考虑了短时域动态学习。例如，MemoryAgentBench（Hu et al., 2025）包含"测试时学习"（Test-Time Learning, TTL）维度，评估智能体在单个交互会话中直接从对话中学习新任务的能力。在实践中，TTL 通过两类任务进行评估：多分类和推荐。在这些设置中，智能体必须利用先前提供的信息——如上下文中的标注样本或长篇电影相关对话历史——来执行将句子映射到类别标签或推荐相关电影等新任务。这评估了持续交互中的即时适应性和知识获取。

**短时域自适应评估的指标与方法** 短时域自适应评估的主要指标和方法旨在量化适应能力（Adaptivity），包括：（1）按迭代步骤统计的成功率（Success Rate by Iteration Steps）（Hu et al., 2024c; Wang et al., 2024j; Zheng et al., 2025b），追踪智能体与环境更多交互或多次尝试任务时性能的改进；（2）学习曲线分析（Learning Curve Analysis），可视化性能（如成功率、准确率）在有限训练步数、回合数或交互次数中的变化（Hu et al., 2024c; Wang et al., 2024j）；（3）适应速度（Adaptation Speed）（Wang et al., 2023a），衡量智能体在短时域内达到特定性能阈值或收敛到最优策略的速度。

短时域自适应评估非常适合评估自进化智能体的初始学习能力和即时适应性。它们能有效地展示智能体是否能够从近期经验中学习并在域内任务上改进性能。该类别广泛应用于当前的自进化智能体。然而，有限的时间窗口使其难以评估长期知识保留（缓解灾难性遗忘）以及跨截然不同或顺序呈现任务的真正终身学习能力。

### 7.2.3 长时域终身学习能力评估

长时域终身学习能力评估对于真正评估自进化智能体至关重要，因为它们关注智能体在多样化环境和长期时间内持续获取、保留和复用知识的能力。如表 9 所示，评估主要侧重于在长时间跨度内跨多样化且可能不断变化的任务或环境流进行持续学习、知识积累和持续性能。这是一个新兴但关键领域，其独特挑战包括灾难性遗忘、跨不同任务的知识稳健迁移、长期高效的资源管理，以及在持续评估演化数据分布时缓解数据泄露。专门的基准测试正在涌现以应对这些复杂挑战。

目前此类基准测试较少。LTMBenchmark（Castillo-Bolado et al., 2024）是专注于长期记忆（Long-Term Memory, LTM）评估的专业基准测试。它通过动态对话测试评估 LLM 智能体的记忆保留和持续学习，使用交错对话配合受控干扰来模拟现实世界的召回挑战。关键指标包括任务准确率、以记忆跨度加权的 LTM Score，以及用于跨架构比较的效率指标（测试/小时、成本）。LifelongAgentBench（Zheng et al., 2025b）是另一个专门设计用于评估智能体终身学习的开创性基准测试。它构建跨数据库（DB）、操作系统（OS）和知识图谱（KG）等领域的相互依赖任务序列，要求智能体逐步建立在先前获得的技能之上。这允许系统地追踪性能改进和跨长期学习轨迹的知识保留。为了解决缺乏多样化环境测试泛化能力的问题，AutoEnv（Zhang et al., 2025i）引入了一个从可分解规则分布自动生成异构世界的框架。该工作还贡献了 AUTOENV-36 数据集以系统地测量智能体的跨环境学习和适应能力。此外，还有一种解决方案通过持续更新基准测试数据集（White et al., 2024; Yang et al., 2025c）或通过重构原始基准测试使基准测试本身演化来构建动态基准测试，以评估自进化智能体，这在一定程度上可以缓解数据泄露（Chen et al., 2025g）。Benchmark Self-Evolving（Wang et al., 2024e）提出了一种通过迭代持续更新现有基准测试的解决方案。类似地，TRACE 框架（Guo et al., 2025a）通过测试时探索使任务演化到更高难度来解决基准测试饱和问题。这些更新、更复杂任务的有效性通过"通过复现验证"范式来保证，确认智能体记录的轨迹可复现。来自此类动态基准测试场景的初步发现表明，随着基准测试的演化，模型性能可能会下降，突显了持续适应的困难。

长时域终身学习的指标超越简单的成功率，用以量化智能体的演化能力，如遗忘率（Forgetting, FGT）、反向迁移（Backward Transfer, BWT）（Zheng et al., 2025c）和单位收益成本（Cost-per-Gain）。长期泛化指标可能涉及评估在持续演化的分布外任务集上的性能，或衡量智能体在跨多个领域长期学习后仍能有效执行的任务广度。

长时域终身学习能力评估对于全面评估自进化智能体的核心承诺至关重要：它们在长期时间内持续学习、保留知识和有效泛化的能力。它们对于评估知识保留（Retention）、向真正新颖场景的泛化（Generalization）以及长期运营效率（Efficiency）至关重要。该领域仍是自进化智能体评估研究的关键前沿。除固定的长时域流之外，开放性变体还会持续演化任务/工具/环境。

### 7.2.4 标准化评估协议

为解决现有设置的异质性并使以轨迹为中心的评估可复现，我们将上述考量进一步提炼为短时域和长时域评估的标准化协议，如表 10 所示。对于短时域设置，我们假设跨任务无状态持久化，并施加每任务进化预算 $K_{\text{short}}$（以迭代次数、工具调用次数、token 数量或挂钟时间计），要求每迭代记录提示词、模型/工具版本、完整推理和动作轨迹及成本明细，并通过按迭代成功率和学习曲线下面积（AULC）报告适应能力（Adaptivity），同时附带基本效率指标（如每次成功消耗的 token 数、延迟）。相比之下，长时域协议假设模型参数、提示词、记忆和工具集跨任务完全持久化，同时指定阶段级和累积进化预算（$K_{\text{stage}}$, $K_{\text{total}}$）以及明确的记忆/工具增长策略，并要求更丰富的日志记录：可复现的轨迹、持久化检查点、定期保留探测、演化决策日志以及人在回路统计，以揭示自我主导的程度。长时域的主要指标则超越即时成功率，扩展到包括保留能力（FGT/BWT 和遗忘曲线）、时间和聚类分布外泛化能力、随时间的单位收益成本和效率漂移，以及长期安全指标（如演化行为下的安全事故率和策略合规性）。我们的实践案例 EvoAgent（Yuan et al., 2025a）说明了标准化协议的可行性和当前局限：它已满足长时域协议的关键要素——持久化的世界模型和经验更新、明确的每子任务步数上限以及对成功率（SR）/探索效率（EE）和挂钟效率的报告——但许多建议的维度（如标准化的保留指标、明确的 CPG/token 漂移和长期安全漂移追踪）未被报告，突显了未来自进化智能体评估需要填补的具体空白。
