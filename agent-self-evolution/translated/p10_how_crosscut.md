5.4 跨维度进化维度

在概述了三种核心进化范式之后，本节，我们来分析它们之间的跨维度特征——揭示不同的设计选择如何在反馈类型、数据源和学习稳定性之间取得平衡。智能体自进化是一个由多个跨维度特征所构成的多方面过程，这些特征塑造了智能体如何在时间进程中学习、适应和提升。除却单一的学习算法或监督信号之外，这些维度定义了自主智能体设计与分析的核心原则。在本节中，我们系统性地从多个关键维度对自进化方法的主要家族——基于奖励的、基于模仿/示教的和基于种群的——进行比较，涵盖学习范式（在线与离线）、策略一致性（on-policy 与 off-policy）以及奖励粒度（基于过程的、基于结果的或混合的）。此外，我们还着重强调其他维度，包括反馈类型、数据源、样本效率、稳定性与可扩展性，如表 4 所示。这一全面比较为理解不同智能体进化方法的优势、局限性与设计权衡提供了统一的视角。

5.4.1 在线学习与离线学习

自进化智能体设计中的另一个基本维度是学习范式，大致可分为离线或在线两类。这一区分取决于智能体的进化更新是在静态的、预先收集的经验数据集上进行（离线），还是通过与实时环境的持续、直接交互来完成（在线）。
离线学习 在离线学习范式中，学习阶段与实时任务执行相分离。离线过程通常包含离线数据生成、筛选和模型微调的循环，侧重于在部署之前构建一个强大且通用的基础模型。该领域的一种主要策略是 LLM 自举（LLM bootstrapping），即模型利用自身生成的内容来增强自身能力。例如，Self-Instruct（Wang et al., 2022）展示了语言模型如何通过生成新指令并将其与自身响应配对，来自举自身的指令遵循能力，从而创建用于微调的综合数据集。在此基础上，WizardLM（Xu et al., 2024a）展示了如何逐步提升这些自生成指令的复杂性，推动模型在更具挑战性的任务上提升能力。尽管这些方法主要通过合成启发式方法聚焦于广泛的能力扩展，扮演着自举阶段的角色，但它们为我们的框架所定义的闭环、经验驱动的进化奠定了必要的基础。在 GUI 和 Web 智能体背景下，离线学习通常涉及利用预先收集的高质量轨迹进行监督微调（SFT）。OS-Genesis（Sun et al., 2024b）引入了一种逆向任务合成方法，用于自动创建轨迹。类似地，UI-Genie（Xiao et al., 2025a）采用统一的奖励模型进行轨迹评估，并结合自改进循环来迭代生成高质量轨迹。两种方法都专注于策划一个丰富的 SFT 数据集，以增强智能体解决复杂任务的能力。除 SFT 之外，离线方法也包含在智能体-环境交互的静态数据集上执行的强化学习。例如，GUI-R1（Luo et al., 2025b）和 InfiGUI-R1（Liu et al., 2025e）利用基于规则的奖励，并在离线 GUI 数据集上应用 R1-style（Guo et al., 2025b）训练。
在线学习 相比之下，在线学习使智能体能够在与实时或模拟环境交互的过程中持续学习和适应。每个动作的反馈用于实时更新智能体的策略、计划或知识库。这使得智能体能够更好地适应动态或未见过的情境。部分智能体通过在线进化并非通过更新模型权重，而是通过即时完善其计划与技能库。例如，Voyager（Wang et al., 2023a）提出了一个由 LLM 驱动的智能体，通过持续探索来学习玩 Minecraft，生成自身的任务课程，并从直接经验中构建持久的技能库。AdaPlanner（Sun et al., 2023）专注于在任务内调整其计划：生成初步计划，从环境接收反馈，并在线完善计划。类似地，SwiftSage（Lin et al., 2023）以快速-慢速思考过程运行，当其快速、直觉模式出现失败时可以进行反思，并切换到更为审慎的使用工具的慢速模式，根据任务难度在线调整策略。强化学习（Reinforcement Learning）是实现在线学习的基本机制，使智能体能够从环境奖励信号中学习。DigiRL（Bai et al., 2024）展示了如何在野外使用自主 RL 训练设备控制智能体，而 DistRL（Wang et al., 2024g）提出了一个异步分布式框架，使此类设备端训练变得可行。MobileGUI-RL（Shi et al., 2025b）通过引入综合任务生成流水线结合基于轨迹感知的奖励的群体相对策略优化（GRPO），来解决在线移动环境中训练 GUI 智能体的特定挑战。

5.4.2 On-policy 与 Off-policy 学习

前一节考察了数据收集与学习的时序（在线与离线），本节则聚焦于智能体进化的策略一致性维度——即智能体是从与其正在改进的策略相同的策略所生成的经验中学习（on-policy），还是从由不同策略生成的经验中学习（off-policy）。这一区分对于理解智能体如何利用其经验数据以及在进化过程中管理学习稳定性与样本效率之间的权衡至关重要。
On-policy 学习 On-policy 方法要求智能体仅从当前策略生成的经验中学习，确保策略一致性，但通常以样本效率为代价。Reflexion（Shinn et al., 2023）通过其迭代自我反思机制诠释了这种方法。智能体使用当前策略生成响应，接收关于失败情况的反馈，并立即将该反馈纳入其推理过程中以进行下一轮迭代。GRPO（Shao et al., 2024b）和 DAPO（Yu et al., 2025a）延续了这一路径，展示了多次采样的有效性。智能体始终从当前行为中学习，保持严格的策略一致性。在智能体设置中，on-policy 方法提供了出色的学习稳定性，避免了困扰 off-policy 方法的分布不匹配问题。然而，这些方法面临样本效率低的问题，因为每次策略更新都需要新鲜的数据收集，在生成高质量轨迹代价高昂的复杂多步推理或工具使用场景中，计算成本很高。
Off-policy 学习 Off-policy 方法允许智能体从由不同策略生成的经验中学习，包括旧版本、其他智能体或人类示教，在显著提升样本效率的同时需付出潜在分布不匹配的代价。Yuan et al.（2024c）展示了一种复杂的 off-policy 方法，其中模型 $M_{t+1}$ 从前一版本 $M_t$ 生成的偏好数据中学习。该系统通过 DPO 内置的 KL 散度约束来处理分布偏移，防止新策略偏离数据生成策略过远。Yuan et al.（2023）展示了另一种强大的 off-policy 范式，通过基于排序的监督从多样的响应来源中学习——包括其他模型、人类和不同的采样策略。该方法巧妙地将对齐问题视为排序问题加以处理，从而优雅地回避了分布偏移，无需要求策略一致性。Zhao et al.（2025b）展示了多智能体设置中的 off-policy 学习，智能体从包含由先前策略版本生成的成功交互轨迹的"经验库"中学习，从而能够高效地复用代价高昂的多智能体协作数据。在智能体设置中，off-policy 方法在样本效率方面表现出色，使智能体能够利用历史数据、专家示教和跨智能体学习。它们在成功轨迹稀少且生成代价高昂的多步推理场景中以及工具使用场景中特别有价值，智能体无需重复环境交互即可从多样的执行示例中学习。然而，这些方法面临分布偏移、奖励黑客（智能体利用训练与部署策略之间的不一致性进行 exploit）等方面的挑战，并需要仔细的正则化以维持训练稳定性。

5.4.3 奖励粒度

奖励设计中的另一个关键选择是其粒度，它决定了智能体在何种细节层次上接收学习信号。奖励粒度的范围从粗粒度的基于结果的奖励（评估整体任务完成情况）到细粒度的基于过程的奖励（评估智能体轨迹中的每一步）。当前的自进化框架采用不同的粒度层次，以根据任务复杂性和期望的学习成果来定制反馈机制。
基于结果的奖励（Outcome-based Reward） 基于结果的奖励是一种反馈机制，基于预定义任务的成功完成来评估智能体。该奖励仅由智能体轨迹的最终状态决定，不考虑中间步骤。在 Web 或 GUI 导航等动态环境中，一个核心挑战是如何有效地从成功轨迹和数量远多于其的失败轨迹中学习。为解决这一问题，直接偏好优化（Direct Preference Optimization，DPO）（Rafailov et al., 2023）被设计为直接最大化偏好响应的似然，同时最小化与参考策略的 KL 散度。类似地，RRHF（Yuan et al., 2023）采用排序损失方法，通过对响应概率进行排序来使多个响应的模型概率与人类偏好对齐，无需辅助的价值模型。此外，多项工作开发了专门基于结果奖励的智能体自进化框架。一个直接的方法是拒绝采样微调（rejection sampling finetuning），如 AutoWebGLM（Lai et al., 2024）中所使用的。该方法使用预设计的奖励模型评估轨迹结果，识别成功轨迹，并用这些高质量数据更新模型。DigiRL（Bai et al., 2024）将 GUI 导航任务建模为马尔可夫决策过程（MDP），并使用基于 VLM 的评估器在回合结束时获得一个最终的、稀疏的奖励。WebRL（Qi et al., 2024）开发了一个稳健的基于结果监督的奖励模型（ORM），以解决动态 Web 环境中固有的反馈稀疏性问题。ORM 在自进化课程框架内评估任务成功与否，使智能体能够从不成功的尝试中学习并逐步改进。
基于过程的奖励（Process-based Reward）与仅提供单一延迟信号的基于结果的奖励相反，基于过程的奖励范式通过评估智能体轨迹中的每一步来提供更精确和细粒度的反馈。过程监督奖励模型（Process-supervised Reward Models，PRMs）已被证明显著优于结果监督奖励模型（Outcome-supervised Reward Models，ORMs），尤其是在需要复杂推理的领域（如求解数学问题）（Lightman et al., 2023）。然而，获取此类细粒度的步骤级反馈传统上需要大量的人工标注，既费时又昂贵，难以扩展。为解决这一标注瓶颈，Math-Shepherd（Wang et al., 2023b）提出了一个自动过程标注框架，利用蒙特卡罗树搜索（MCTS）通过评估每个步骤导出正确最终答案的潜力来收集步骤级监督。类似地，AlphaMath（Chen et al., 2024a）训练了一个价值模型来评估解路径中步骤的正确性，并在 MCTS 框架内通过探索与利用来同时更新策略和价值模型。通过利用基于过程的奖励，智能体能够以渐进、逐步的方式提升其能力。rStar-Math（Guan et al., 2025）和 AgentPRM（Choudhury, 2025）都提出了迭代演化策略和过程奖励模型的方法，生成无需人工标签的逐步更高质量的推理路径。Agent Q（Putta et al., 2024）将逐步验证机制集成到其 MCTS 流程中，以收集高质量轨迹，然后通过 DPO 训练迭代地完善策略。
混合奖励（Hybrid Reward） 混合方法旨在通过同时纳入最终任务成功的清晰性（基于结果）和中间步骤的细粒度指导（基于过程），来提供更全面的学习信号。这些方法克服了仅基于结果的信号的稀疏性，同时将智能体的逐步推理锚定在终极任务目标上。例如，GiGPO（Feng et al., 2025a）通过引入双层奖励机制来解决训练长时域智能体的不稳定性问题。它基于整个轨迹的最终成功提供回合级奖励，同时为中间动作分配局部化的步骤级奖励。这种双重信号同时提供了高层方向性目标和低层纠正性指导。类似地，SPA-RL（Wang et al., 2025d）提出了一种奖励分解方法，弥合了稀疏的结果信号与密集的过程反馈之间的鸿沟。它根据最终任务完成情况，将增量进步归因于多步轨迹中的每一步，从而有效地将基于结果的奖励分配到整个过程中。这种方法产生了密集的中间过程奖励，增强了强化学习的有效性，同时保持了与终极任务目标的对齐。

5.5 自进化方法的其他维度

除学习范式、策略一致性和奖励粒度这些核心维度之外，表 4 还突出了区分自进化方法的其他几个重要维度：
反馈类型。反馈的性质差异很大：基于奖励的方法利用标量奖励、自然语言信号或模型置信度；模仿方法侧重于示教轨迹和推理过程；基于种群的方法使用适应度分数或竞争信号。反馈类型从根本上决定了智能体用来改进的信息是什么。
数据源。基于奖励的方法通常通过智能体-环境交互或工程化规则生成数据，而模仿学习往往依赖人类或专家生成的示教。基于种群的方法从多个智能体或代际的集体经验中汲取营养，实现了多样性探索，但需要显著的协调。
样本效率。模仿学习的样本效率通常最高——前提是有高质量的示教可用，因为智能体可以直接模仿专家行为。基于奖励的方法效率适中，且效率高度依赖于奖励的稀疏程度。基于种群的进化往往样本效率较低，因为它通常需要通过大量试验来评估大量的智能体变体。
稳定性。基于奖励的学习对奖励函数的质量和设计敏感，存在奖励黑客或意外行为的风险。模仿学习在很大程度上取决于示教的质量和多样性。基于种群的方法对种群规模和多样性敏感，种群规模过小或同质化程度高则面临早熟收敛的风险。
可扩展性。可扩展性取决于数据或反馈收集的可行性以及学习的并行化能力。当反馈实现自动化时（例如通过模拟器），基于奖励的方法具有良好可扩展性。模仿学习往往受制于收集示教的成本。基于种群的方法可以扩展到大规模计算，但高度消耗资源。
综合来看，这些维度为自进化策略提供了一个更加细致、多维的视角，帮助实践者为各自领域面临的挑战选择和设计最为匹配的智能体学习流水线。
