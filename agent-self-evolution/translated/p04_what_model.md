# 3   进化什么？

自进化智能体（self-evolving agent）与静态智能体的区别不在于其包含的组件，而在于其内部状态能够根据自身的轨迹、反思和反馈信号进行自主修改。因此，本节的核心问题在于识别智能体系统 Π = (Γ, {ψi }, {Ci }, {Wi }) 中的进化位点（evolutionary loci）——即系统中能够以经验驱动且持久的方式重写状态的各个部分，从而实现累积性的自我提升。

根据第 2.1 节的形式化描述，这些进化位点对应于智能体系统的四大支柱。我们的研究从智能体的认知核心——即模型（Models）{ψi }——开始，其参数可以通过自我生成的监督信号、执行轨迹或环境反馈进行持续更新（Zhou et al., 2025e; Wang et al., 2025q）。随后，我们考察上下文（Context）{Ci } 的演化——包括指令（Xiang et al., 2025; Khattab et al., 2023）和长期记忆（Chhikara et al., 2025; Wang et al., 2024j）——它在智能体进行反思、存储和检索经验的过程中不断演变，从而塑造未来的决策行为。基于这一内部基础，我们进一步考察工具（Tools）{Wi } 的演化，智能体根据可验证的交互信号自主创建（Qiu et al., 2025b）、精炼（Qu et al., 2025）和管理（Wang et al., 2025k）可执行技能。最后，我们扩展至智能体架构（Agentic Architecture），系统的架构（Hu et al., 2024c; Zhang et al., 2024c）和协作结构（Wan et al., 2025）随时间优化，实现超越单个组件的结构性适应。我们以表 2 展示这些进化位点的代表性示例。

## 3.1   模型（Models）

模型是自进化的主要位点，因为其参数能够根据智能体自身的轨迹、反思和交互结果进行自主重写。这些模型通过持续调整内部参数并扩展其功能能力来实现进化的能力，对于开发自主、通用型智能体至关重要。与依赖大量人工标注数据集和固定训练范式的静态系统不同，自进化模型能够通过交互、自我监督的数据生成和动态学习循环来提升，从而实现更高的效率、适应性和可扩展性。

具体而言，我们阐述了模型演化展开的主要维度。这包括从自我生成的监督信号中学习以精炼模型权重，通过与构建环境或外部环境的交互进行演化，以及整合能够直接重塑未来推理行为的反馈信号。这些策略共同代表了一种从被动学习范式向主动自我提升的转变。

**策略（Policy）** 自进化智能体可以精炼其参数以更好地完成目标任务。传统方法在工具使用基准测试中收集训练智能体的数据成本高昂，且往往覆盖范围有限，而纯合成数据生成流程通常存在质量不足的问题。因此，近期研究强调使智能体能够自主生成数据以改进自身的模型权重。一种代表性的方法是自挑战智能体（Self-Challenging Agent, SCA）（Zhou et al., 2025e），其中语言模型在生成可执行 Code-as-Task 问题的挑战者角色和执行者角色之间交替。该模型随后使用成功解决方案产生的轨迹对参数进行微调，在复杂多步骤任务上取得了显著的性能提升。类似地，自我奖励自我改进框架（Self-Rewarding Self-Improving framework）（Simonds et al., 2025）实现了内部自我评判机制，使模型能够自主生成问题、解决问题并评估自身表现，从而产生无需外部标注的自包含微调数据。该方法在复杂推理任务中展现了显著的改进。除任务创建外，另一个有前景的研究方向是直接将交互反馈用于参数更新。例如，SELF（Lu et al., 2023）、SCoRe（Kumar et al., 2024）和 PAG（Jiang et al., 2025b）将执行轨迹或自然语言批评解释为在线监督微调（Supervised Fine-Tuning, SFT）结合强化学习（Reinforcement Learning, RL）框架中的奖励信号，实现策略的持续改进。TextGrad（Yellamraju et al., 2024）进一步拓展了这一概念，将非结构化文本反馈视为可微分的训练信号，能够直接影响提示设计（prompt design）和模型参数。此外，AutoRule（Wang & Xiong, 2025）将语言模型的推理轨迹和偏好反馈转换为显式基于规则的训练奖励，通过结构化奖励信号提升模型输出的质量。总体而言，这些进展描绘了一条清晰的轨迹——从智能体自主构建训练任务，到直接根据执行反馈精炼自身参数，凸显了模型通过从自身产生的数据中学习来实现持续进化的能力。

**经验（Experience）** 智能体不仅可以通过调整内部参数进行演化，还能够积极地与环境交互乃至构建自身所处的环境，获取经验，并将其转化为驱动迭代改进的学习信号。这一环境循环为智能体提供了可扩展自我适应所需的复杂性和多样性。自挑战智能体（SCA）（Zhou et al., 2025e）在任务层面体现了这一动态特性，智能体自主生成新的 Code-as-Task 问题、执行这些问题，然后筛选成功的轨迹用于自我再训练。AgentGen（Hu et al., 2024b）将这一概念扩展至完整环境生成，从初始语料库合成多样化的模拟世界（采用 PDDL 或 Gym 风格的格式）。它实现了一个双向演化循环，逐步调整任务难度，使智能体能够在动态结构化的课程中持续成长。Reflexion（Shinn et al., 2023）通过引入自我反思机制对此进行了补充，智能体迭代记录对其先前行动的自然语言批评，引导未来行为以避免重复的错误。此外，AdaPlanner（Sun et al., 2023）引入了闭环自适应规划，使智能体能够根据环境反馈实时精炼策略，有效地对行动序列进行即时调整以应对即时结果。类似地，Self-Refine（Madaan et al., 2023b）采用迭代精炼循环，智能体反复批评并修订其初始输出，在无需显式再训练的情况下显著提升了任务准确率。SICA（Self-Improving Coding Agent，自改进编码智能体）（Robeyns et al., 2025b）进一步突破了边界，使智能体能够自主编辑其底层代码和工具，通过直接的自我修改迭代增强其核心推理能力。从强化学习的视角来看，RAGEN（Wang et al., 2025q）和 DYSTIL（Wang et al., 2025c）等框架将多步骤工具使用任务概念化为马尔可夫决策过程（Markov Decision Processes），通过丰富的环境奖励和策略归纳循环优化智能体策略。RAGEN 利用来自环境的密集反馈迭代微调行动策略，而 DYSTIL 利用语言模型生成的高层策略建议，逐步将复杂决策技能内化到强化学习智能体中。总体而言，这些方法突出了一种引人注目的范式：自进化智能体不仅利用自我生成的数据，而且积极重塑其环境和内部机制以驱动持续学习。这种动态交互循环指向了深深根植于经验适应的自主、开放式的改进循环。
