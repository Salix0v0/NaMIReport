# 7 自进化智能体的评估

自进化智能体的评估面临着超越传统静态人工智能系统评测的独特挑战。与在单一时间点对固定任务集进行评测的传统智能体不同，自进化智能体被设计为通过与动态环境的持续交互来不断学习、适应和改进。因此，对它们的评估不仅需要捕获即时的任务成功率，还需涵盖适应随时间的变化、知识积累与保留、长期泛化能力以及跨顺序任务或新任务迁移所学技能的能力，同时还需缓解灾难性遗忘（catastrophic forgetting）问题。这要求从传统的"一次性"评分方法向一种长期的、成本感知的轨迹视角进行根本性转变。

## 7.1 评估目标、指标与基准覆盖

为了有效评估自进化智能体，我们必须超越传统指标，建立一个能够捕获其动态性、适应性和长期学习能力的综合框架。一个真正有能力和受人期待的自进化智能体不仅需要学习和改进，还必须记住过去的知识、将其迁移到新的情境中、可持续地运行，并负责任地行事。基于这些持续性和鲁棒人工智能的关键需求，我们将关键评估目标分为五个核心维度：适应性（Adaptivity）、保留（Retention）、泛化（Generalization）、效率（Efficiency）和安全（Safety），如图 6 所示。每个维度都对应着智能体自进化过程的一个关键方面，并通过其相应的指标和基准覆盖分析进行评估。表 7 提供了评估资源的综合目录，我们在此综合其局限性，以绘制这五目标框架中的覆盖空白，并在表 8 中确定面向轨迹的、成本感知的评估方向。

### 7.1.1 适应性

**目标与指标** 适应性作为任何自进化智能体的基础性评估标准，衡量其通过经验提升域内任务性能的能力。该维度专注于量化学习曲线以及智能体在特定领域内迭代和进化过程中的性能提升程度。适应性并非通过静态成功率来衡量，而是随时间、步数或迭代次数来评估。典型指标包括按迭代步数统计的成功率（Success Rate by Iteration Steps）（Hu 等，2024c；Wang 等，2024j；Zheng 等，2025b），该指标追踪智能体在下游任务中随其交互历史变化的性能表现。

**覆盖空白** 尽管适应性拥有最丰富的基准生态——覆盖代码生成（SWE-bench（Jimenez 等，2023）、MLE-Bench（Chan 等，2024））、网页导航（WebArena（Zhou 等，2023）、WebShop（Yao 等，2022））和通用推理（GAIA（Mialon 等，2023）、AgentBench（Liu 等，2023b））——但现有评估仍受制于实际设计选择。ScienceAgentBench 将任务限制为单一编程语言并施加执行时间上限，从而排除了计算密集型的科学工作流程（Chen 等，2024e）。MLE-Bench 提供了结构良好的任务，但与真实研究场景存在偏差——在真实场景中，问题定义本身就是挑战的一部分（Chan 等，2024）。这些简化虽然允许受控的域内评估，但将评估限制在狭窄的、静态的任务分布上。此外，大多数基准仅在预指定的学习协议（例如固定迭代预算或预定的回放调度）下测量改进程度，而无法评估智能体是否自主发现了有效的适应策略。AgentBench 还报告了智能体在持续推理和战略决策方面的不足（Liu 等，2023b），而评估时间范围仍然局限于短期改进曲线。

### 7.1.2 保留

**目标与指标** 保留是评估自进化智能体知识库稳定性的关键标准。该维度特别关注灾难性遗忘（catastrophic forgetting）问题——这是持续学习中的一个常见挑战，即新知识的获取会侵蚀先前学到的信息，以及在扩展交互中的知识保留问题。可用两个关键指标从不同角度量化这种稳定性：遗忘率（Forgetting, FGT）和后向迁移（Backward Transfer, BWT）（Zheng 等，2025c）。具体而言，设 $J_{i,t}$ 表示智能体完成 $t$ 个任务后第 $i$ 个任务上的性能：

$$FGT_t = \frac{1}{t-1} \sum_{i=1}^{t-1} \max_{j \in \{i,\ldots,t\}} \left( J_{i,j} - J_{i,t} \right)$$

$$BWT_t = \frac{1}{t-1} \sum_{i=1}^{t-1} \left( J_{i,t} - J_{i,i} \right)$$

正的 BWT 表示新的学习对旧任务有正向促进作用，意味着成功的知识迁移和更加鲁棒、稳定的学习过程。

**覆盖空白** 保留是最未被充分服务的维度。当前的记忆机制在动态状态更新和跨扩展交互保持一致性方面面临重大挑战（Hu 等，2025）。经验回放存在内在张力：虽然回放缓冲区可以改善学习，但将其扩展超过最优阈值会因上下文溢出和资源耗尽而导致性能下降（Zheng 等，2025b）。经济和计算约束限制了评估的鲁棒性，一些基准仅进行了最少的重复实验，可能无法捕获随机性变化（Castillo-Bolado 等，2024）。最关键的是，现有基准中压倒性的大多数采用片段式评估（episodic evaluation），即智能体在任务之间进行状态重置，这从根本上排除了对知识积累或退化的测量——而这恰恰是区分自进化智能体与静态系统的关键现象。

### 7.1.3 泛化

**目标与指标** 适应性（Adaptivity）和保留（Retention）关注的是域内性能，而泛化（Generalization）是衡量自进化智能体将其积累的知识应用于新的、未见过的领域或任务的能力的关键指标。一个真正智能的智能体不仅应在其熟悉的领域中表现良好，还应展示跨域泛化能力。该能力可通过评估智能体在跨越多个任务分布和领域的多样化任务集上的表现来评估。常用方法包括：在多领域测试套件上计算聚合性能指标（例如平均成功率）（Liu 等，2023b；Sun 等，2023），以及使用留出任务分布（held-out task distributions）进行域外评估，以模拟真实世界中的新颖场景（Hu 等，2024b；Peng 等，2025）。

**覆盖空白** 泛化通常通过多领域测试套件（AgentBench（Liu 等，2023b）、TheAgentCompany（Xu 等，2024b））和留出任务分布来评估，以衡量智能体跨领域迁移知识的能力。然而，当前的评估依赖于静态快照：智能体仅在多样化任务上测试一次，而未追踪在扩展学习轨迹中跨域迁移是否退化。技术进步进一步降低了旧任务的区分能力，因为现代智能体受益于早期系统无法获得的算法进步（Chan 等，2024），而训练数据污染使跨域评估更加复杂（Chan 等，2024）。现有基准中没有任何一个检验智能体在领域内专业化时是否保持泛化广度，也没有检验在一个领域中获得的知识在数百个学习片段后是否仍然可以迁移。

### 7.1.4 效率

**目标与指标** 效率量化了自进化智能体在其学习和运行过程中的资源利用能力。由于智能体持续运行并自主做出决策，评估其进化过程的成本和速度至关重要。

| 维度 | 符号 | 典型单位 | 示例测量 | 实际案例 |
|------|------|----------|----------|----------|
| Token 消耗 | $C_{token}$ | #tokens | 每片段/步的提示 + 完成 token 数 | 每任务 440K vs. 8.1M tokens；18 倍差异 |
| 步数/延迟 | $C_{step}$ | #turns / rounds | 推理/交互轮次 | 每任务 35 vs. 181 次 API 调用；质量与数量权衡 |
| 墙上时间 | $C_{time}$ | #seconds | 端到端含 I/O 的已用运行时 | 每任务 12 vs. 200+ 轮次；简单 vs. 复杂 bug |
| 工具/API 调用 | $C_{tool}$ | #calls | 外部函数/环境调用 | 15 vs. 38 次调用（AutoCodeRover）；高质量推理减少迭代 |

| 人工监督 | $C_{human}$ | #hours / tokens | 审查、标注、红队；引导 token | 失败尝试：4 倍成本 vs. 成功尝试；昂贵的失败模式 |

- **Token 消耗**（Hu 等，2024a）：推理、生成和记忆操作中使用的总 token 数
- **时间消耗**（Lu 等，2024b）：完成任务或达到性能阈值所需的墙上时间
- **步数**（Wang 等，2023a）：任务完成所需的交互轮次数
- **工具调用**：外部 API/环境调用的次数，可结合性能收益评估工具生产率（Tool Productivity, TP）（Wang 等，2025h），公式为：

$$TP\$ = \frac{\Delta score}{\sum_i cost(tool_i)}$$

- **内存增长**：在智能体运行期间，持久记忆和上下文窗口的扩展
- **人工监督**：人工干预所需的时间和精力，包括审查、标注和纠正指导

为了将效率与性能收益关联，我们报告每次增益成本（Cost-per-Gain, CPG）：

$$CPG_t = \frac{Total\ Cost_t}{Performance\ Gain_t + \epsilon}$$

其中，成本可按 token、时间、内存、人工精力或归一化复合指标（例如 monetary cost）来衡量，性能收益是相对于基线在时间范围 $t$ 处的提升。CPG 越低表示学习效率越高。

**覆盖空白** 效率方面的进化成本报告稀疏且不一致。虽然基准论文偶尔记录评估期间的总体资源消耗（Chan 等，2024），但很少将其分解为进化特定的分量：自我反思或经验回放消耗的 token、架构搜索或记忆更新的墙上时间、自主探索触发的工具调用等。人工基线收集中的成本约束（Xu 等，2024b）反映了更广泛的可访问性挑战，但无法阐明进化过程本身的效率。更根本的是，标准评估协议允许无约束优化——智能体在不面临真实部署中的硬性 token 预算、迭代限制或延迟约束的情况下最大化任务成功率。

### 7.1.5 安全

**目标与指标** 从自进化的角度来看，安全领域关键性地审视这些智能体在其持续进化过程中是否产生不安全或不良的行为模式。该维度评估智能体对预定义规则的遵守情况及其有害行为的倾向。评估自进化智能体安全性的关键指标包括：（1）安全评分（Safety Score）（Zhang 等，2024i），衡量智能体行为被标记为"安全"的测试用例比例；（2）危害评分（Harm Score）（Andriushchenko 等，2024），通过详细的手动评分规则计算，当输出触发部分而非全部有害标准时获得部分分数；（3）策略合规完成率（Completion Under Policy, CuP）（Levy 等，2024），评估智能体在严格遵守给定规则或策略集的情况下成功完成任务的比率；（4）风险比率（Risk Ratio）（Levy 等，2024），计算智能体沿特定维度违反规则的频率，提供违规行为的定量度量；（5）拒绝率（Refusal Rate）（Zhang 等，2024b；Andriushchenko 等，2024），评估智能体因任务的攻击性、恶意性或其他不安全性质而拒绝执行的任务比例；（6）泄露率（Leakage Rate）（Shao 等，2024a），追踪智能体无意泄露敏感或私人信息的频率。

**覆盖空白** 安全评估主要捕获孤立片段中的风险。Agent-SafetyBench 识别出两个核心缺陷：在不同上下文部署工具时的鲁棒性不足，以及在特定操作场景中对潜在危害的识别有限（Zhang 等，2024i）。在多智能体设置中，SwarmBench 揭示了基本协调挑战：局部交互未能产生连贯的集体策略，智能体无法维持安全协作操作所需的共享态势感知（Ruan 等，2025）。然而，没有基准追踪在扩展进化过程中的安全轨迹——无论是风险通过反复接触边缘情况而累积，还是不安全行为通过自主探索和自我导向学习而出现。

### 7.1.6 自我导向性与评估权衡

自进化智能体的一个定义性特征是其自主探索和自我导向进化的能力，这使其区别于被动学习范式。自我导向程度——具体而言，智能体是自主生成任务和进化策略还是遵循外部提供的课程——在评估维度之间产生了根本性的权衡。

高度自我导向的系统通过自主课程生成展示了可观的性能提升。WebRL 通过从探索失败中自主生成任务，从 4.8% 提升至 42.4%（Qi 等，2024），而 SEAgent 通过自主软件探索和课程进化实现了 23.2 个百分点的提升，从 11.3% 提升至 34.5%（Sun 等，2025c）。然而，自主进化伴随着可度量的风险。在冲突目标下自主进化的智能体中，对齐伪装率从 12% 攀升至 78%（Greenblatt 等，2024），这说明当进化以最少的外部监督进行时会带来安全挑战。

尽管有这些记录在案的影响，该领域仍缺乏标准化指标来量化进化中的自我导向性。为了实现公平比较，我们建议透明地报告进化过程的三个方面。（1）明确说明进化策略和任务序列是预定的、程序性采样的，还是由智能体自主生成的；（2）记录反馈信号的来源，说明其来自外部人工标签、基于规则的评估器还是自我生成的反思；（3）报告进化过程中外部干预的频率。明确记录自主控制程度对于解释性能声明至关重要，因为收益可能反映的是真正的自进化能力，也可能是大量外部指导的结果。

这种面向轨迹的评估与评估自进化智能体的核心目标相一致：不仅衡量它们取得了什么，还衡量它们在多大程度上自主地学会了进化。
