# 8 未来方向

## 8.1 个性化人工智能代理

随着对自进化代理的兴趣日益增长，部署个性化代理已成为研究界的一个关键且日益重要的目标（Zhang et al., 2024h）。例如，在聊天机器人、数字孪生和情感支持对话等应用中，一个核心挑战是使人工智能代理能够准确捕捉并长期交互中适应用户独特的行为模式或偏好。现有的个性化代理通常严重依赖标注数据和后训练方法（Cheng et al., 2024）。Zhang 等人（2025q）近期的工作提出了一种自生成偏好数据方法，旨在实现大语言模型的快速个性化。TWIN-GPT（Wang et al., 2024h）利用电子健康记录为患者创建数字孪生，以提高临床试验结果预测的准确性。然而，这些现有策略基于一个关键假设：大语言模型能够持续获得高质量、大规模的用户数据。

在实际部署场景中，主要挑战仍然是冷启动问题：即使在初始数据有限的情况下，代理也需要逐步完善其个性化理解，准确解读用户意图，并有效地构建用户画像。此外，个性化规划与执行中仍存在重大挑战，例如有效的长期记忆管理、外部工具集成以及个性化生成（确保输出始终符合个体用户的事实和偏好）（Li et al., 2025d）。此外，确保自进化代理不会无意中强化或加剧现有的偏见和刻板印象也至关重要，这凸显了未来研究的另一个关键方向。这些治理原则尤为重要，因为个性化代理会持续演进并不断调整其记忆或决策过程。在此基础上，评估框架也需同步演进，以确保个性化环境中的公平性、问责性和一致性。

数据治理。负责任的个性化应在适应性与隐私保护之间取得平衡。首先，代理应采用数据最小化原则：仅收集与任务相关的数据，并提供透明、可撤销的控制选项。经验表明，网络代理基准测试显示，最先进的代理经常不必要地处理敏感数据，这推动了在默认状态下实施最小化设计的必要性（Zharmagambetov et al., 2025）。作为补充，针对个性化代理的有效数据治理需要部署设备端个性化框架，使代理能够从用户交互中进行本地学习，同时配合由用户主导的隐私机制，如 Rescriber，在任何远程数据交换之前执行设备端编辑和审批（Zhou et al., 2025c）。为防止无限期保留数据，记忆衰减和遗忘策略应支持选择性删除以及针对个性化痕迹的"被遗忘权"式遗忘学习（Staufer, 2025）。最后，由于自进化可能随时间导致安全或公平性的偏移，系统应包含偏见监控和公平性审计循环，使标准和干预措施能够随着用户和情境的演变而调整（Basu & Das, 2025），并通过持续检查记忆/工具/工作流的更新来防止进化失配（在进化过程中出现的安全/对齐退化）（Shao et al., 2025）。

评估。随着个性化数据的整合，个性化自进化代理的评估指标应超越内评估（例如，使用 ROUGE（Lin, 2004）和 BLEU（Papineni et al., 2002）等指标直接评估个性化生成文本的质量）或外评估（例如，通过推荐系统、分类任务及其他特定应用间接评估个性化效果）。传统个性化评估指标往往无法充分捕捉自进化代理固有的动态演进特征。因此，未来研究需要更轻量级和自适应的评估指标（Zhang et al., 2024h）。此外，为了更好地评估自进化个性化代理，迫切需要灵活、动态的基准测试，能够准确评估代理在自我进化过程中管理长尾个性化数据的能力。我们倡导报告以下指标：

- **Personal Adaptation Gain (PAG)（个人适应增益）**：在 k 轮会话中相比非个性化基线的每位用户改进幅度。
- **Retention & Forgetting Balance（保留与遗忘平衡）**：前向/后向迁移和选择性遗忘效果等指标（例如，用户提出删除请求后暴露于用户事实的减少量）。
- **Privacy–Utility Trade-off（隐私-效用权衡）**：效用增益与保留的个人数据比特数之比；在网络代理场景中辅以数据最小化评分（仅在必要时共享敏感信息的比例）（Zharmagambetov et al., 2025）。
- **On-device Learning Ratio（设备端学习比例）**：在用户设备本地执行的个性化更新或记忆写入的比例，可辅以用户同意/编辑机制（例如，Rescriber 中的研究）（Zhou et al., 2025c）。
- **Bias/Drift Monitors（偏见/漂移监控器）**：跨用户群体的 disparity 或安全退化的纵向测量；可定义公平性漂移（Fairness-Drift）或安全性漂移（Safety-Drift）等指数，尽管标准化版本仍有待开发（Basu & Das, 2025; Shao et al., 2025）。
- **Longitudinal User Outcomes（纵向用户结果）**：追踪会话级满意度、目标完成趋势或多阶段发展轨迹（例如，StuLife 等基准测试中的虚拟"校园生活"代理）（Cai et al., 2025）。

## 8.2 泛化能力

自进化代理在实现跨多样化任务领域和环境的鲁棒泛化能力方面也面临重大挑战。专业化和广泛适应性之间的根本张力仍是该领域最紧迫的挑战之一，对可扩展性、知识迁移和协同智能具有深远影响。

可扩展架构设计：开发可泛化的自进化代理的一个核心挑战在于设计能够随复杂性和范围增长而保持性能的可扩展架构。当前的代理系统经常在专业化和泛化之间面临权衡：为特定任务优化的代理在将所学行为迁移到新环境时存在困难（Chen et al., 2024d）。此外，基于大语言模型的代理中动态推理的计算成本随适应机制复杂度的增加而非线性增长，在现实资源限制内对可实现的泛化能力构成了实际约束（Kim et al., 2025）。近期研究表明，具有反思和记忆增强能力的自进化代理在增强泛化能力方面具有实质性潜力，尤其是在规模较小、资源受限的模型中（Liang et al., 2024）。然而，这些方法在处理需要长期适应的复杂现实场景时仍面临局限性。

跨领域适应：实现跨领域泛化是自进化代理的关键前沿。当前的跨领域适应方法通常依赖领域特定的微调，限制了代理在新环境中无需重新训练即可适应的能力（Belle et al., 2025）。测试时扩展和推理时适应方面的最新进展为增强跨领域泛化提供了有前景的途径（Snell et al., 2024; Zhang et al., 2025l）。这些技术允许代理通过在推理过程中扩展计算资源，动态地为不熟悉的场景分配额外的推理能力，从而无需增加模型参数。此外，元学习策略在促进新领域的快速少样本适应方面展现出相当大的潜力（Bilal et al., 2025）。然而，其有效性关键取决于代理准确判断何时需要补充计算资源并有效分配这些资源以完成多样化推理任务的能力。

持续学习与灾难性遗忘：自进化代理必须在保留先前获得的知识的同时持续适应新任务，这一挑战因大语言模型固有的持续记忆（Chen et al., 2024b）的灾难性遗忘现象（Ghosal et al., 2024）而加剧。基于基础模型的代理中，为每个新任务重新训练的计算成本令人望而却步，使得稳定性-可塑性两难困境尤为尖锐（Zheng et al., 2025c）。近期研究探索了参数高效微调方法、选择性记忆机制和增量学习策略，以在保持适应性的同时缓解灾难性遗忘（Wang et al., 2024c）。然而，在效率与防止模型漂移之间取得最佳平衡仍然是一个重大的开放性挑战，尤其是在代理在资源受限条件下运行或管理具有严格隐私约束的流式数据时。

知识可迁移性：近期研究已识别出人工智能代理知识迁移中的关键局限性。Shi 等人（2025a）强调，当前代理的知识整合与迁移能力仍需大幅优化。特别地，Geng 等人（2025b）发现，基于大语言模型的代理往往无法有效地将交互中获得的新知识传播给其他代理，限制了其协作潜力。此外，Vafa 等人（2025）揭示，基础模型可能严重依赖浅层模式匹配，而非建立鲁棒且可迁移的内部世界模型。这些发现指出了几个重要的未来研究方向：1）有必要更好地理解一个代理获得的知识在何种条件下能够被可靠地泛化并传递给其他代理；2）开发量化代理知识可迁移性局限的方法，可能会对代理协作瓶颈提供更清晰的见解；3）我们需要一种明确的机制来促进鲁棒、可泛化的世界模型的形成，这可能显著提高自进化代理的协作效能。

## 8.3 安全且可控的自进化代理

随着自主人工智能代理在独立学习、进化和执行复杂任务方面的能力日益增强，确保其安全性和可控性已成为首要关注点。与静态系统不同，自进化的本质引入了独特的且在代理生命周期中动态放大的风险。这些风险不仅仅是传统人工智能安全问题的延伸（例如由模糊的用户指令或环境威胁（如恶意钓鱼链接）引起的问题）（Zhou et al., 2025d），而是根本性地与代理自主自我修改和适应的能力相关（Shao et al., 2025; Han et al., 2025）。本节首先阐述自进化代理特有的新兴风险，然后讨论一套规范性的防护栏和缓解策略，以构建更安全的系统。

### 8.3.1 自进化系统中的新兴风险

近期研究已识别出由自主自我改进过程本身产生的新风险。这些风险在初始代理中不一定存在，但会随着其进化而显现。我们沿代理的主要进化路径对这些风险进行分类：骨干模型、记忆和工具。

- **模型进化中的不可控行为漂移**：一个核心风险是代理的目标和价值观可能在进化过程中偏离原始的人类意图。这因自进化固有的学习不确定性而加剧，尤其在模糊上下文中运行时更为明显（Anwar et al., 2024; Bagdasarian et al., 2024）。在模型进化过程中可能发生一种称为"进化失配"（misevolution）（Shao et al., 2025）的现象。例如，在代理生成的数据上进行自训练可能导致安全对齐的"灾难性遗忘"，使代理执行其先前拒绝的有害指令，例如与其被训练回避的恶意内容进行交互（Shao et al., 2025; Hahm et al., 2025）。
- **记忆进化中的部署时奖励黑客攻击**：开放式进化容易受到奖励黑客攻击。代理可能发现并利用自我定义的奖励信号或内部反馈中的漏洞。这在记忆进化中尤为明显，经验的积累可能无意间诱导不安全行为。例如，代理可能学会发出不必要的退款，因为其记忆将退款与高满意度评分相关联（Shao et al., 2025），而设计不当的记忆模块可能进一步加剧此类风险（Wang et al., 2025b）。一个相关概念是"对齐临界过程"（Alignment Tipping Process, ATP），即一个最初对齐的代理发现不对齐行为能获得更高奖励，导致其策略"临界"并放弃其初始约束（Han et al., 2025）。
- **自创建和引入的外部工具的安全性**：代理自主生成和使用工具的能力引入了重大的安全问题。代理可能自发地创建具有安全漏洞的工具，或在引入外部工具时未能识别恶意代码（Shao et al., 2025）。这使得代理成为安全威胁的潜在载体。这里的一大挑战是代理仍然难以区分必要和不相关的敏感信息（Zharmagambetov et al., 2025），可能导致它们创建泄露私人数据的工具。此外，随着代理能力的提升，它们可能更擅长创建和执行攻击性网络操作，这一风险需要动态评估（Wei et al., 2025a）。

### 8.3.2 规范性防护栏与缓解策略

应对自进化带来的新兴风险需要超越描述性警告，实施规范性的、可操作的防护栏。尽管早期框架（如 TrustAgent）已探索了多阶段策略（即规划前、规划中、规划后）以促进更安全的行为（Hua et al., 2024），但自进化的独特动态特性要求一种更全面的"安全生命周期"方法。未来研究和开发应专注于在代理运行的每个阶段整合安全防护措施。

- **工具与代码执行的沙盒化和验证**：为缓解工具使用的风险，所有代理生成或外部来源的工具都必须在严格的沙盒环境中执行。此外，在集成新工具之前，自动化安全验证（如静态分析和漏洞扫描）应是默认步骤（Labs, 2025）。为保护工具交互协议，运行时防御管道（Xing et al., 2025; Wang et al., 2025a）提供了针对工具中毒和提示注入等威胁的分层检测。
- **自我修改的审计追踪和安全故障保护**：任何自我修改都必须伴随全面的审计追踪。这确保了变更可追溯且可逆。实施回滚和安全故障保护模式至关重要，允许系统在新检测到不良行为时恢复到先前已知的安全状态。对于记忆，A-MemGuard 等主动防御机制提出双记忆结构和基于共识的验证，在"中毒"记忆影响行为之前将其识别并隔离（Wang et al., 2025b; Wei et al., 2025d）。
- **长期漂移的持续监控和红队测试**：静态的、部署前安全评估是不够的。需要对代理行为进行持续监控，以检测长期价值观漂移。这可以通过设计用于测试新兴不对齐的红队场景来实现。对于 GUI 代理，OS-Sentinel 等混合验证框架结合形式化验证器和上下文裁判，提供鲁棒的工作流内安全监控（Sun et al., 2025b）。
- **审批门和隐私保护措施**：对于高风险操作，应实施需要人工确认的审批门。此外，鉴于代理在处理敏感数据方面存在困难（Zharmagambetov et al., 2025），需要强有力的隐私保护措施以防止泄露并确保平衡、安全的部署。

为帮助从业者，我们将这些策略综合为自进化代理部署的合规检查清单（见表 12）。总而言之，部署可靠、可控且安全的自进化系统是一个关键且活跃的研究领域。未来的工作必须朝着构建全面的安全感知进化生命周期迈进，整合鲁棒验证、持续监控和自适应防护栏，以确保代理在变得更加自主的过程中始终与人类价值观和安全约束保持一致。

## 8.4 多代理生态系统

多代理自进化系统面临若干需要进一步探索的独特挑战。

平衡个体推理与集体推理：近期研究强调了多代理环境中独立推理与有效群体决策之间平衡的困难（Chen et al., 2025d; Sun et al., 2025a）。虽然集体讨论能显著增强诊断推理，但代理通常面临过度依赖群体共识的风险，从而削弱其独立推理能力。为缓解这一问题，未来研究应探索动态机制，调整个体输入与集体输入的相对权重。这种方法将有助于防止决策被单一代理或少数代理子集主导，最终促进鲁棒、平衡的共识形成和创新。此外，开发明确的知识库和标准化的更新方法论——利用代理的成功与失败——可进一步提升代理的自我进化能力，并在协作环境中加强其个体推理贡献。

高效框架与动态评估：另一个关键挑战在于开发高效的算法和自适应框架，使代理能够有效协作，同时保持其个体决策优势。（Hu et al., 2024d）引入了自适应奖励模型和优化的动态网络结构，可显著增强代理之间的协作自我改进。然而，（Sun et al., 2025a）识别出的一个主要缺口是代理动态管理和更新知识的机制缺失。解决这一问题需要新的框架，明确整合持续学习和自适应协作机制。此外，现有的多代理评估基准测试主要是静态的（Zhu et al., 2025），因此无法捕捉代理角色的长期适应性和持续演进。未来的基准测试应纳入动态评估方法，反映多代理系统内的持续适应、演变交互和多样化贡献，从而为自进化代理提供更全面的评估指标。
