## 3.3 工具（Tools）

智能体的能力从根本上由其所能运用的工具定义。智能体发展轨迹中有一个关键转折：从仅仅作为工具的使用者，转变为能够自主制造工具的主体。这种从依赖预定义静态工具集到使智能体能够自主扩展和改进自身技能的转变，是迈向认知自给自足的关键一步。在这一范式下，智能体能够动态地调整自身能力，从而解决初始设计者未曾设想的复杂长尾问题。这一演化进程在三个相互关联的维度上展开：工具的发现、掌握与管理，具体如下文各小节所述。

### 自主发现与创造（Autonomous Discovery and Creation）

自主工具创造的首要动力在于克服固定工具集固有的局限性，使智能体能够在需要时灵活创新。当前的 方法已从机会主义式的发现延伸至形式化的合成。在一端，以 Voyager 为代表的智能体通过尝试—错误的涌现式探索来构建不断扩展的技能库，其驱动来自于在 Minecraft 等复杂开放环境中探索的内在动机。这种探索式方法在产生多样化技能方面表现强劲，但可能缺乏精确性。相比之下，ATLASS、Alita 和 Live-SWE-Agent 等系统采取更具反应性的方法，通常从零开始创建新工具，或在发现能力缺口时即时利用检索增强生成（RAG，Retrieval-Augmented Generation）技术在开源代码仓库中进行搜索。在另一端，是高度结构化的框架，将工具创造视为一项审慎的工程过程。例如，CREATOR 将抽象工具创造（例如推理一个用于对 N 天平均温度的可复用函数的一般结构）与具体工具使用（例如决定如何将该函数应用于特定城市和时间范围）分离开来，从而增强了模块化程度和可复用性。更为形式化地，SkillWeaver 分析成功的人类或智能体任务轨迹，提出、综合并精炼新技能为稳健可复用的 API，确保更高的初始质量。此外，CRAFT 等框架表明，为特定领域创建专用工具集对于补充通用模型至关重要，能够实现专家级性能而不牺牲适应性。RL-GPT 将生成的代码实现集成到强化学习流程中，将其作为工具来处理复杂任务，同时采用"策略即代码"（Code-as-Policy）方法直接处理简单任务。这种集成根据环境反馈动态地适应和演化，实现持续改进。然而，这种日益增长的自主性也引入了重大挑战，尤其是在安全与安保方面。无约束的代码生成存在创建具有可利用漏洞或意外有害行为的工具的风险，使得自动验证和沙盒化成为未来研究的重点领域。

### 通过迭代精炼实现掌握（Mastery Through Iterative Refinement）

自我创造工具的激增必然要求一种稳健的掌握机制；一个新生成的工具通常只是一个脆弱的脚本，而非可靠的功能函数。这正是迭代精炼变得不可或缺之处。LearnAct 和"从探索到掌握"（From Exploration to Mastery）等框架建立了一个关键的自我修正循环，智能体从中学习自身经验。这涉及到解决困难的"信用分配"（credit assignment）问题：精确地确定哪一行代码或哪个参数导致了失败。为此，智能体分析多种丰富的反馈信号——包括编译器错误、意外的 API 返回值、环境状态变化，甚至来自用户后续行为的隐含信号。目标不仅是调试工具底层代码，还要精炼其文档（例如其 docstring 和参数描述），这对提高智能体未来理解和正确使用该工具的能力至关重要。这种精炼过程也为有价值的人机协作提供了可能。虽然完全自主是最终目标，但许多系统可以设计为"人在循环中"（human in the loop）的模式，人类专家可以提供修正建议、高层建议，或验证新创建的工个。这种协作方式可以显著加速掌握过程，并确保智能体的技能与人类意图和安全标准保持一致。归根结底，这种自我打磨的过程将一个初生的技能提升为可信的能力，确保智能体不断增长的技能库不仅在数量上增长，更重要的是在质量和稳健性上不断增长。

### 可扩展的管理与选择（Scalable Management and Selection）

当智能体掌握的技能库扩展到数百乃至数千个时，它面临"丰裕诅咒"（curse of abundance）。挑战从创建工具转变为高效地管理和从中选择。庞大的库创造了巨大的搜索空间，使得传统检索方法既缓慢又不准确。为克服这一问题，ToolGen 通过将工具编码为语言模型词表中独特的标记，实现了根本性的范式转变。这优雅地将工具检索重新定义为生成问题，利用 Transformer 强大的模式识别能力来预测最合适的工具——作为思维过程自然的延续。TOOLMEM 使智能体能够将不同工具的优势和劣势学习并存储在专用记忆中。在推理时，智能体检索这些知识以做出更明智的决策，针对特定任务需求优化工具选择。除了选择单一工具外，高级智能体还须擅长工具组合——学习将多个工具以新颖的序列链接起来，以解决多步骤问题。这是一项更高阶的管理任务。AgentSquare 等架构式方法进行一种元学习，自动搜索智能体的模块化设计空间——包括其规划、记忆和工具使用组件——以找到复杂任务执行的最优配置。作为这一演化趋势的逻辑终点，Darwin Gödel Machine 等远见概念提出了开放-ended 进化的框架，智能体可以在其中从根本上重写自身的核心代码。在这一愿景下，智能体与其工具之间的界限变得模糊，导致超越工具增强本身的递归式自我改进 cascade。实际上，这条演化路径的整个目标在于建立一个封闭且良性的循环：一个真正自主的智能体，能够感知自身能力的缺口，创造新颖的解决方案，通过实践掌握它们，并将它们无缝集成到一个连贯管理且不断扩展的技能库中。

## 3.4 架构（Architecture）

下一代智能体系统的定义性特征是其内在的自我改进能力。这标志着从具有固定能力的系统到能够自主提升自身性能的系统的根本性转变。通过将自身的内部逻辑和协作结构视为可优化的组件，这些系统能够根据反馈调整自身行为和设计，达到静态设计无法企及的效率和效能水平。本节详细阐述这种自我优化是如何实现的：首先考察单智能体系统内部的改进，然后探索复杂多智能体系统的协同演化。

### 3.4.1 单智能体系统优化（Single-Agent System Optimization）

**LLM 调用节点优化（LLM-Invoking Node Optimization）** 在孤立状态下优化单一 LLM 调用是直接的，但在智能体系统中，它变成了一个困难的信用分配问题，因为任何单一变化的效果都会被后续步骤所掩盖。研究通过使节点级组件可优化来应对这一挑战，主要遵循两种策略。第一种策略专注于在固定智能体拓扑内精炼节点。一个典型例子是 TextGrad（Yellamraju et al., 2024），它受反向传播启发，使用"文本梯度"将来自最终输出的反馈向后传播至整个工作流程，引导每个节点进行系统性的局部精炼，而不改变系统的整体结构。第二种并行策略将这种组件级优化直接集成到系统架构本身的搜索中。在这一方法下，节点特征成为更大搜索空间中的可调参数。例如，框架可以将提示工程直接嵌入搜索循环，使系统能够同时发现最优工作流程和每个智能体最有效的指令（Zhou et al., 2025a）。类似地，EvoFlow（Zhang et al., 2025c）使用进化算法，通过从多样化池中选择最适合每个任务的 LLM 来构建异构工作流程。这种整体性策略使得发现同时为结构和单个智能体能力进行协同优化的系统成为可能，有效地平衡了整体性能和成本等指标（Ye et al., 2025a）。

**自主智能体优化（Autonomous-Agent Optimization）** 在优化单个 LLM 调用节点的基础上，更深层次的自我改进将自主智能体作为整体实体来对待。这一演化沿两个主要方向推进：优化智能体的高层架构设计，以及使智能体能够直接修改自身的源代码。第一种方法专注于发现最优的智能体结构。AgentSquare（Shang et al., 2025）是其典型代表，它将规划器和记忆模块等组件定义为模块化设计空间，然后使用进化算法为给定任务找到最有效的组合。第二种方向涉及能够动态重写自身操作代码的智能体。这在激进的系统中可见，如 Darwin Gödel Machine（Zhang et al., 2025h）递归地修改自身的 Python 代码库，以及 AlphaEvolve（Novikov et al., 2025）使用进化编码来改进特定算法。类似地，Gödel Agent（Yin et al., 2025）为智能体提供了一个自引用框架来分析和修改其逻辑。这两个方向（优化智能体的架构"蓝图"及其功能代码）共同展示了将智能体基本结构和逻辑转变为可学习组件的关键趋势。MemEvolve（Zhang et al., 2025g）引入了一个元进化框架，它不仅演化智能体的经验记忆，还演化记忆系统本身的架构。通过双层优化过程，它使信息的编码、存储和检索机制适应特定任务领域的需求。

### 3.4.2 多智能体系统优化（Multi-Agent System Optimization）

智能体在系统中的组织方式及其通信方式（系统的拓扑结构）从根本上决定了其解决复杂问题的能力。该领域已从使用固定的、人为设计的通信结构发展到创建动态系统，能够自动将自身组织适应给定任务，使它们能够发现并利用最有效的协作模式。这一演化沿两个主要方向展开：静态显式工作流程的优化，以及动态内部策略的协同演化。

**智能体工作流程优化（Agentic Workflow Optimization）** 智能体工作流程的优化专注于为给定问题找到最有效且往往是静态的通信和任务分配结构。早期研究奠定了重要基础，AutoFlow（Li et al., 2024d）等研究展示了从自然语言自动创建线性工作流程，GPTSwarm（Zhuge et al., 2024）提出了统一的基于图的框架。同时，其他基础性工作探索了智能体如何通过符号学习将交互经验提炼为显式可解释的逻辑规则来指导未来决策（Zhou et al., 2024b）。将系统抽象为可调组件——无论是节点、边还是符号规则——是至关重要的。然而，这些早期系统往往缺乏一种正式的方法来高效地探索大量可能的配置和交互空间。

重大突破出现在 ADAS（Hu et al., 2024c）和 AFlow（Zhang et al., 2024c）将这一挑战正式定义为搜索和优化问题之时。ADAS 通过将系统设计视为在基于代码的配置所构成的图灵完备空间中的搜索，确立了理论愿景。在此基础上，AFlow 通过引入代表常见智能体模式的可复用算子，并采用蒙特卡洛树搜索（MCTS，Monte Carlo Tree Search）来高效地探索巨大的设计空间，使其变得实用。这些工作共同确立了将智能体系统设计视为可处理的优化问题的核心方法，证明了自动发现的工作流程可以优于人工设计的。

在这一形式化之后，研究迅速向为每个特定查询创建定制化智能体系统的方向多样化发展。主要出现了两种策略：基于搜索的生成和基于学习的生成。MaAS（Zhang et al., 2025d）等基于搜索的方法创建潜在架构的"超网"（super-net），然后从中采样专用系统。并行地，基于学习的方法训练模型直接生成有效拓扑。ScoreFlow（Wang et al., 2025n）使用新颖的偏好优化方法训练生成器，而 FlowReasoner（Gao et al., 2025）使用强化学习训练一个元智能体来即时构建定制工作流程。这一针对特定查询的生成研究方向持续活跃（Ye et al., 2025b; Ke et al., 2025）。此外，重要的是注意这一过程并不局限于拓扑本身；许多这些框架同时进行节点级优化，例如将提示协同优化或将异构模型选择作为架构生成过程不可或缺的组成部分（Zhang et al., 2024c; Zhou et al., 2025a; Zhang et al., 2025c）。

所有搜索和学习方法面临的一个关键挑战是评估每个潜在工作流程的计算成本（Shang et al., 2025）。为此，研究人员开发了轻量级预测模型。Agentic Predictor（Trirat et al., 2025）是一个典型例子，它训练一个模型基于结构和语义特征准确估计工作流程的性能，而无需完整执行。通过提供快速且经济的评估代理，这些预测器显著加速了优化过程，使探索庞大设计空间变得可行（Zhang et al., 2025s）。

**多自主智能体优化（Multi-Autonomous-Agent Optimization）** 与优化系统的显式工作流程结构不同，这一研究方向专注于多个自主智能体如何通过交互协同演化其内部行为策略。这种方法使协调、任务分配和有益竞争等涌现能力成为可能。例如，ReMA（Wan et al., 2025）使用多智能体强化学习（MARL，Multi-Agent Reinforcement Learning）协作训练一个高层元思考者和一个低层执行器，显著提升了推理基准测试的性能。在此基础上，GiGPO（Feng et al., 2025b）通过聚合轨迹来提供更精确的信用分配，提升了长时域任务的成功率。为支持这一方向，MARTI（Liao et al., 2025）等平台提供了开源基础设施来编排和扩展这些语言模型集体的训练。总体而言，这些研究强调多智能体强化学习是培养群体级能力的有效路径，这些能力是单个智能体单独无法实现的。
