## 3.2 上下文（Context）

上下文是 LLM 智能体中被进化的核心组成部分，它决定了智能体的行为方式。首先，我们需要厘清"提示优化"（prompt optimization）和"记忆进化"（memory evolution）这两个在不同文献中使用的术语。在大多数情况下，这两个术语可以互换使用，因为它们都指代上下文窗口中所包含的内容。提示优化关注的是"如何措辞或组织指令能使 LLM 表现更好"，侧重于措辞表述和排列顺序等细节；而记忆进化则关注"如何存储、遗忘和检索上下文，使智能体能够保持信息获取并更好地执行任务"，其核心是什么过去的信息需要被呈现或归档。

### 3.2.1 记忆进化（Memory Evolution）

基于 LLM 的智能体正越来越多地被设计配备长期记忆机制，这些机制随着智能体不断解决问题并与其环境交互而成长和适应（Shan et al., 2025; Qian et al., 2023a）。进化的记忆使智能体能够积累知识、回忆过去的事件，并根据经验调整其行为。许多研究强调，有效的记忆管理对智能体性能至关重要（Zhong et al., 2024; Zhang et al., 2025e; Yan et al., 2024）。SAGE（Liang et al., 2024）利用艾宾浩斯遗忘曲线来决定记忆什么或遗忘什么。A-mem（Xu et al., 2025a）通过动态索引和链接来更新智能体的记忆结构，创建互联的知识网络，遵循 Zettelkasten 方法的基本原则。Mem0（Chhikara et al., 2025）引入了一个两阶段流程，智能体首先从最近的对话中提取突出事实，然后决定如何更新长期记忆：智能体可以添加（ADD）新事实、合并/更新（MERGE/UPDATE）冗余事实，或删除（DELETE）矛盾信息。

此外，Memory-R1（Yan et al., 2025）提出了一个强化学习框架，用于训练一个专门的记忆管理器智能体（Memory Manager agent），使其学习选择结构化操作如添加、更新和删除。这种机制确保了智能体的长期记忆保持一致性和时效性。MemInsight（Salama et al., 2025）通过语义结构增强原始记忆，对过去的交互进行摘要和标记，以便后续检索。REMEMBER（Zhang et al., 2024a）将 LLM 与经验记忆相结合，利用强化学习信号来决定每轮交互后如何更新该记忆。Memento（Zhou et al., 2025b）通过采用在线强化学习来优化案例检索策略，使智能体能够在不微调 LLM 参数的情况下实现持续适应，从而从存储在不断演变的记忆库中的过去经验中学习。MemGen（Zhang et al., 2025f）引入了一种在潜在空间（latent space）中运行的动态生成记忆，它使用一个学习到的记忆触发器来决定何时调用记忆，并使用编织器（weaver）来构建潜在 token 序列，实现推理与记忆的流畅交织。

记忆进化的一个关键方面是使智能体能够从过去的经验中学习启发式方法或技能。先进的智能体不仅检索完全匹配的历史案例，还将经验提炼为更一般的指导原则（Zhao et al., 2024a; Fu et al., 2024）。Expel（Zhao et al., 2024a）处理过去的轨迹以生成洞察和规则，用于指导进一步的交互。这种经验知识的积累带来了可衡量的收益，随着经验的增加，智能体表现稳步提升。ReasoningBank（Ouyang et al., 2025）进一步发展了这一思想，从成功和失败的经验中提炼出可泛化的推理策略，存入结构化记忆中。该研究还引入了记忆感知的测试时扩展（memory-aware test-time scaling），为每个任务生成多样化的经验。

其他系统侧重于存储问题解决的高层构建模块。例如，Agent Workflow Memory（Wang et al., 2024j）记录常见的子任务序列（工作流），使得解决复杂任务的智能体能够检索并重用一个已验证的动作序列，而非从头开始规划。类似地，MUSE（Yang et al., 2025a）为长时域任务引入了一种经验驱动的智能体，围绕一个分层记忆模块展开，该模块将经验组织为战略性记忆、程序性记忆和工具使用记忆。智能体通过"计划-执行-反思-记忆"（Plan-Execute-Reflect-Memorize）循环来充实这一记忆库，从而能够在工作中学习。在 Richelieu 外交智能体中，系统通过自对弈游戏来增强其记忆，从而改进谈判策略，将模拟交互中产生的洞察存储起来，以优化未来的决策（Guan et al., 2024）。通过将具体的情节泛化为可重用的知识，这些方法说明了记忆进化如何将智能体的一次性经验转化为长期能力，从而推动智能体的进化。

### 3.2.2 提示优化（Prompt Optimization）

如果说记忆进化关注的是智能体保留什么知识，那么提示优化（Prompt Optimization, PO）则使 LLM 智能体能够通过精炼输入骨干模型的指令来实现自进化，这些指令直接改变模型的行为而不修改模型权重（Ramnath et al., 2025）。早期研究将指令设计视为一个搜索问题。APE（Zhou et al., 2022）生成候选提示，在验证样本上评分，并选择最佳的。ORPO（Yang et al., 2023）通过让模型迭代地重写自己的提示来扩展这一思路，并以先前输出的反馈作为引导。ADO（Lin et al., 2025a）引入了 DSP，对迭代提出的提示施加语义约束，以促进找到最优提示。ProTeGi（Pryzant et al., 2023）生成自然语言"修正"，这些修正作为编辑操作应用于提示，形成一种文本形式的梯度下降。PromptAgent（Wang et al., 2023d）将提示发现建模为蒙特卡洛树搜索（Monte-Carlo Tree Search），策略性地探索指令空间，而像 PromptBreeder（Fernando et al., 2023）这样的进化方法则维护一个种群以发现越来越有效的指令。REVOLVE（Zhang et al., 2024e）通过跟踪模型响应的轨迹并应用平滑更新，进一步稳定了长时间优化过程的运行。将这种自主性推向极致，SPO（Xiang et al., 2025）创建了一个完全自包含的循环，模型生成自己的训练数据，并通过对输出的成对偏好比较来精炼提示，无需任何外部标注数据或人类反馈。总体而言，这些技术表明智能体可以自主地改进其提示策略，将提示文本变为一个可学习的组件，与智能体的经验共同进化。

为了解决某些优化器的 brevity bias 和上下文崩溃问题，ACE（Agentic Context Engineering）（Zhang et al., 2025m）将上下文视为随着时间积累策略的综合 playbook。它使用模块化的智能体流程，通过增量更新来演化上下文，同时支持离线提示优化和在线记忆适应。

在复杂系统中，智能体通常需要编排一系列 LLM 调用或与其他智能体协作，这使得提示设计成为一个多节点问题。DSPy 等框架将整个工作流程表示为一张图，其子提示针对全局目标进行联合调优（Khattab et al., 2023）。Trace（Wang et al., 2023c）、TextGrad（Yellamraju et al., 2024）和 LLM-AutoDiff（Yin & Wang, 2025）将这一思想泛化，将每个提示视为可微调程序中的一个参数，并传播自然语言"梯度"来精炼每个步骤。在协作场景中，多智能体系统搜索（Multi-Agent System Search, MASS）（Zhou et al., 2025a）首先优化个体角色提示，然后精炼智能体间的通信模式，而 MAS-ZERO（Ke et al., 2025）则为每个新问题动态地提出和修订角色提示，以组建一个高效的团队。像 EvoAgent（Yuan et al., 2024b）和 AgentSquare（Shang et al., 2025）这样的进化系统将每个智能体及其提示作为模块，采用变异和选择来发现超越手工设计的表现更优的专业化团队。这些方法将提示优化从单一指令扩展到定义整个工作流程或智能体社会的语言。
